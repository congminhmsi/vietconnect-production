üîß Troubleshooting

Wallet Connection Not Working

Check the following:

  • ‚Ä¢ Ensure NEXT_PUBLIC_WALLET_CONNECT_PROJECT_ID is correctly set in .env
  • ‚Ä¢ Verify the Project ID is valid and active in your Reown dashboard
  • ‚Ä¢ Confirm the frontend has been restarted after adding the environment variable
  • ‚Ä¢ Check browser console for any error messages

Modal Displays Fallback Project ID Warning

If you see a fallback Project ID being used:

  • ‚Ä¢ The environment variable is not properly configured
  • ‚Ä¢ Double-check the variable name is exactly NEXT_PUBLIC_WALLET_CONNECT_PROJECT_ID
  • ‚Ä¢ Ensure the .env file is in the root directory of the project
  • ‚Ä¢ Restart the application after making changes

Network Issues

If users can't connect to specific networks:

  • ‚Ä¢ Verify the network is configured in frontend/config/wallet.ts
  • ‚Ä¢ Check if the user's wallet supports the selected network
  • ‚Ä¢ Ensure the network configuration includes correct chain IDs

üîí Security Considerations

Best Practices
  • ‚Ä¢ Keep Project ID Secure: While the Project ID is public-facing, it's tied to your domain. Configure domain restrictions in Reown dashboard for added security.
  • ‚Ä¢ Use Environment Variables: Never hardcode the Project ID directly in your source code.
  • ‚Ä¢ Monitor Usage: Regularly check your Reown dashboard for unusual activity or usage patterns.
  • ‚Ä¢ Domain Whitelisting: In production, configure allowed domains in your Reown project settings to prevent unauthorized use.
